# ADR-003: PostgreSQL as Default Relational Database

## Status
Accepted

## Context
Teams were choosing databases per service ad hoc (SQL Server, MySQL, MongoDB), fragmenting
operational knowledge, backup tooling, and licensing.

## Decision
PostgreSQL is the default database for all new services. Deviations (e.g. a document store
for genuinely schema-free data) require a dedicated ADR with justification.

## Rationale
- One engine to operate, monitor, and back up.
- No per-core licensing costs.
- JSONB covers most "we need NoSQL" cases without a second technology.

## Consequences
- Existing SQL Server services are not migrated; the decision applies to new services only.
- Entity Framework Core with the Npgsql provider is the standard data access stack.
