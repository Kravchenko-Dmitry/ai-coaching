# ADR-002: Use Asynchronous Messaging for Order Processing

## Status
Accepted

## Context
Order processing involves long-running workflows (payment, inventory, shipping) where
synchronous REST chains create tight coupling and cascading failures under load.

## Decision
For the order processing domain, services communicate asynchronously via a message broker
(Azure Service Bus). Commands and events are published as messages; REST (per ADR-001)
remains the default for simple request/response interactions outside this domain.

## Consequences
- Order services must be idempotent (at-least-once delivery).
- Correlation ids are mandatory on all messages for tracing.
- Adds Service Bus as operational infrastructure for the order domain only.
